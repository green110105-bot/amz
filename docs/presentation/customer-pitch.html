5 维评分系统（17 子项 · 百分制）

维度权重子项与算法
D1 关键词覆盖度 25% D1.1 主词覆盖（12 分） = 搜索词 Top 10 在标题+五点中出现比例 × 100
D1.2 长尾词覆盖（8 分） = Top 30 长尾词（≥4 tokens）在五点+描述+A+ 中覆盖比例
D1.3 否定词避免（5 分） = 100 − 禁用词出现数 × 20
D2 卖点清晰度 20% D2.1 USP 强度（5 分） LLM 评估五点首句独特性 vs 类目均值
D2.2 实证强度（5 分） 计数数据/认证短语（"5000mAh" "FCC"等）
D2.3 决策因素覆盖（5 分） 5W2H 框架覆盖度 / 7
D2.4 vs 竞品差异化（5 分） 与头部竞品 embedding 距离

D3.2 好评高频卖点突出度（8 分） 自家好评 Top 5 卖点是否突出

M1 实现原理 · 4/5

图像生成三层方案（应对亚马逊审核）

核心挑战：亚马逊主图必须"实物拍摄风格"。纯 DALL-E / Imagen 渲染极易被审核拒（行业拒绝率约 30%）。我们用三层方案降到约 1%。
方案 A · 默认推荐

AI 概念稿 + 用户拍摄

流程：

  • AI 生成 3 个"概念稿"（含布光建议、角标位置、卖点排布）
  • 不直接发布，作为"拍摄简报"交给设计师/拍摄团队
  • 用户拍真实照片后上传，系统做合规检查 + 角标合成

用途：替代昂贵的设计师沟通；将"我想要 X 风格" → 可视化稿

审核拒绝率：约 1%

方案 B · 微改

AI 后期处理（不重新合成产品）

适用：已有合规真实主图，只需微改

  • 仅做后期处理：合成角标、白底替换、亮度对比微调
  • 不允许 AI 重新合成产品本体
  • 多模态合规检查 + 视觉相似度（embedding）双校验
  • 与原图相似度 > 0.85 才允许

审核拒绝率：约 5%

方案 C · 进阶模式

SD + 真实产品 LoRA fine-tune

适用：需要换角度 / 换场景的真实感生成

  • 用户提供 30+ 张产品多角度真实照片
  • 系统训练专属 LoRA（仅该 SKU 可用）
  • 后续生成基于真实产品特征
  • 仍需用户确认"够真实"才发布

仅限"产品已经稳定 + 长期销售"的 SKU 投资

A+ / Gallery 不受此限制

A+ 和 Gallery 允许插画 / 信息图 / 场景图 / 人物，直接 AI 生成无问题。仍走内容合规检查（无医疗声明 / 无第三方品牌 / 无虚假超级化）。

版权与侵权防范

  • 所有"AI 生成"输出附带 generation_metadata（模型 / prompt / 参考图）
  • 与竞品图像 embedding 相似度 > 0.92 → 阻止生成 + 风险提示
  • 用户上传"参考图"必须勾选合法使用权
  • 系统不存储未授权竞品图（短期分析后丢弃）

利润口径 ROAS + 自动执行护栏

双 ROAS 求解（M2 ↔ M3 循环依赖）

M3 用 M2 的"利润 ROAS"做决策，但 M2 利润含 M3 广告分摊。双口径并存避免循环：

Sales ROAS = Sales / Spend # 亚马逊原口径 Profit ROAS = (Sales × Margin − OtherCosts) / Spend # 利润口径 # 求解三阶段 1. 基础口径：M2 出销售 ROAS → M3 粗判 2. 精确口径：M2 用 M3 关键词归因 → 出利润 ROAS 3. 迭代收敛：M3 自动执行 1h 后重算（最多 3 次防抖） # UI 颜色编码 利润 ROAS < 1 → 🔴 红（亏钱跑广告） 销售高利润低 → 🟡 黄（看似在赚 实际没赚） 利润 ROAS > 1.5 → 🟢 绿

自动执行护栏（5 项 Safety Check）

  • 1
    动作类型白名单：bid 调整 ±15% / 加预算 ≤+20% / 减预算 ≤-30% / 加否词。删除 / 暂停 Campaign 须人工
  • 2
    数值阈值：bid_change_max 15% / budget_inc_max 20% / 周累计 50%
  • 3
    每日配额：单 SKU ≤ 5 操作 / 单租户 ≤ 100 操作
  • 4
    SKU 排除：标签"重点 / 测试 / 保护期"自动跳过
  • 5
    时间窗口：Prime Day / 黑五期间不自动

失败熔断：单 SKU 失败 1 次自动模式暂停 4h；失败 3 次 / 24h 全租户全自动停止 + 紧急通知。

一键回滚：所有自动执行 7 天内可一键回滚（反向 API 调用 + 审计记录）。

主权三档：手动（仅给清单，全人工）/ 半自动（一键批量执行，默认）/ 全自动（按规则自动）。所有写操作必经"跨模块审计中心"统一记录，含冲突检测 + 资源锁 + 配额管控。
